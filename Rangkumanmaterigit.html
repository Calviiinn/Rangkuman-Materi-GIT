Rangkuman Materi GIT

Definisi GIT

Git adalah salah satu version control system, Git mencatat setiap perubahan file dicode atau sourcecode.

Manfaat Dari GIT

Bisa menyimpan seluruh versi source code

Berkolaborasi membangun sistem jadi mudah

Bisa berkontribusi di project opensource

Memudahkan dalam tracking perubahan

Memahami cara deploy modern

Kapan GIT Dibutuhkan

Saat ingin membangun sistem dengan kolaborasi

Membuat versi dari sistem

Backup sourcecode

Berkolaborasi dengan programmer dunia

Tools and Program for GIT

Akun GITHUB/GITLAB, Sourcetree, Texteditor,Lightshot, Grepper, Screen O Matic, Zoom

Fundamental GIT

Deskripsinya, perintah-perintah dasar yang sering dilakukan untuk mengoperasikan GIT

Manfaat, terbiasa mengerjakan proyek dengan terdokumentasi, fleksibel dalam mengoperasikan fitur, mempercepat dalam mengerjakan proyek terlebih dalam proyek kolaborasi

Git Command

Git Init, Git Config, Git Status, Git Add, Git Commit, Git Clone, Git Remote, Git Push, Git PUll

Kapan Git Dibutuhkan? Saat mengerjakan proyek.

Fungsi Dari Git Command

Git Init

Digunakan untuk menginisialisasi repository, atau memberi tahu folder saat ini akan dijadikan repository, kita tidak bisa menggunakan Git tanpa menginisialisasikanya.

Git Config

Konfigurasi emall dan username, biasanya disamakan dengan akun gitlab/github, config global untuk konfigurasi ke server, config local untuk konfigurasi ke server, untuk melihat semua konfigurasi yang ada di git bisa menggunakan perintah git config --list.

Git Status

Digunakan untuk mengetahui status file, status file secara umum ada 3; Untracked, staging area, modified

Git Add

Untuk memasukan file ke staging area

Git Log

Untuk melihat riwayat commit, tekan q untuk keluar dari log

Git Commit

Untuk mencatat perubahan yang dilakukan

Git Clone

Digunakan untuk mengcopy repo dari server

Git Remote

Semacam api untuk berinteraksi dari client dengan server

Git Push

Untuk mengirimkan commit ke server

Git Pull

Untuk mengambil commit dari server jika ada perubahan diserver yang dibuat oleh branch lain.

Git Branch

Deskripsi Branch

Cabang / copyan dari repo

Manfaat

Paralell development, bisa digunakan untuk memanajemen pengembangan, misal test branch, unstable branch, stable branch.

Kapan Dibutuhkan

Saat mengembangkan fitur baru

Branch Command

Git branch: melihat semua branch local

Git branch --all: melihat semua branch public

Git branch nama_branch: membuat branch baru

Git checkout nama_branch: untuk berpindah branch

Git branch -d nama branch: untuk menghapus branch

Git merge nama_branch: untuk menggabungkan commit dari dua branch, relatif terhadap branch yang aktif saat ini, untuk melakukan merge harus dipastikan tidak ada file yang belum di commit.

Conflict, kondisi yang terjadi ketika merging 2 branch yang memiliki perubahan pada file yang sama, ketika conflict kita tidak bisa melakukan commit, untuk melakukan commit, conflict harus diselesaikan dulu.

Sourcetree

Sourcetree adalah software berbasis GUI untuk memudahkan dalam mengoperasikan git

Manfaat sourcetree, mudah digunakan, solving conflict jadi lebih mudah, mempercepat kinerja.

Kapan sourcetree dibutuhkan? Setiap saat.
